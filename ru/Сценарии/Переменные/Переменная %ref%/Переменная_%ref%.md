# Переменная %ref%


<p class="yfm-lead">Здесь мы расскажем, как делать реферальные ссылки на базе Smartbot Pro</p>

### Сначала немного теории

{% note info %}

Реферальная ссылка - это ссылка, содержащая информацию, откуда пользователь переходит по ней.&#x20;

{% endnote %}

Например, мы размещаем ссылку в рекламной кампании, и пользователи через нее попадают в нашего бота.&#x20;

Бот считывает ссылку, извлекая параметр %ref%, в котором указан источник перехода.

Описание параметров %ref% и %ref\_source% можно увидеть в разделе "Переменные":

### Как этим пользоваться?

Пишем ссылку следующего вида:

Для групп ВК:

> https://vk.me/группа?ref=&#x20;

Для Telegram:

> https://t.me/канал?start=

Вместо "группа" или "канал" - название вашего сообщества из адресной строки. После знака = укажите латинскими буквами источник, где размещаете эту ссылку, например, reklama.&#x20;

Таким образом, готовая ссылка может выглядеть следующим образом: \
\
https://vk.me/club192634149?ref=reklama\
\
https://t.me/smartbotpro?start=reklama

{% note info %}

Обратите внимание, что в %ref%-параметре не должно быть символов: одиночная скобка ), слеш /

{% endnote %}

После клика по ссылке параметр "reklama" сохранится как пользовательская переменная человека, и в дальнейшем параметр можно будет использовать для условий в сценарии.&#x20;

<figure><img src="https://4058588211-files.gitbook.io/~/files/v0/b/gitbook-x-prod.appspot.com/o/spaces%2FvTd8XmFMqkqZga7zhLuk%2Fuploads%2FnJGKFzQ1wRoVYIasvhDY%2Fimage.png?alt=media&#x26;token=1b8bc40a-aadc-40b9-844b-13f223a088d8" alt=""><figcaption></figcaption></figure>

Если человек перейдет по другой ссылке и перезапустит бота, его реферальный параметр обновится.
