# Блок "Создать сделку"


<p class="yfm-lead">В этой статье мы разберем, для чего нужен этот блок и как с ним работать.</p>
Чтобы отправлять заявки в amoCRM из бота, перейдите в нужный сценарий в конструкторе, нажмите на раздел "Интеграции" и выберите "amoCRM". Нажмите "Создать сделку" - на экране появится блок для отправки данных в amoCRM.&#x20;

### Параметры сделки&#x20;

1. В открывшихся настройках блока выберите **аккаунт в amoCRM\***. \
   \*Подробная [инструкция](https://docs.smartbotpro.ru/scenarii/integracii/amocrm#podklyuchenie-amocrm-k-smartbot-pro) том, как привязать ampCRM к Smartbot.&#x20;

<figure><img src="https://4058588211-files.gitbook.io/~/files/v0/b/gitbook-x-prod.appspot.com/o/spaces%2FvTd8XmFMqkqZga7zhLuk%2Fuploads%2Fg9VMM1jkJorJ9EDSaMys%2Fimage.png?alt=media&#x26;token=9aec2079-afe8-4203-a3f1-32fa3cee73d7" alt="" width="563"><figcaption></figcaption></figure>

2. Выберите **воронку**, в которую нужно добавить сделку, и статус сделки. Эти данные автоматически подтянутся из вашего кабинета amoCRM.&#x20;

<figure><img src="https://4058588211-files.gitbook.io/~/files/v0/b/gitbook-x-prod.appspot.com/o/spaces%2FvTd8XmFMqkqZga7zhLuk%2Fuploads%2FspfJVBATzoXyrcJTesDL%2Fimage.png?alt=media&#x26;token=c759d94c-fb3c-4cbe-8562-2b2d88062c22" alt="" width="563"><figcaption></figcaption></figure>

3. Укажите **название сделки.** По умолчанию это поле заполнено переменной %username%, то есть в качестве названия сделки будет использоваться ник пользователя в социальной сети. Вы можете использовать любое другое значение.  \
   Если оставить поле пустым, название сделки в amoCRM будет иметь следующий формат.&#x20;

<figure><img src="https://4058588211-files.gitbook.io/~/files/v0/b/gitbook-x-prod.appspot.com/o/spaces%2FvTd8XmFMqkqZga7zhLuk%2Fuploads%2FHYw6NI8bAGWsfHsGAuOI%2Fimage.png?alt=media&#x26;token=0174d24f-5df4-4fd2-9d92-a193b6652248" alt="" width="375"><figcaption></figcaption></figure>

4. Выберите **ответственного менеджера**&#x20;

<figure><img src="https://4058588211-files.gitbook.io/~/files/v0/b/gitbook-x-prod.appspot.com/o/spaces%2FvTd8XmFMqkqZga7zhLuk%2Fuploads%2FddHGWRs40UfdDope7Jq5%2Fimage.png?alt=media&#x26;token=fc186cf8-d19f-4fc3-88f0-b83efecd545e" alt="" width="425"><figcaption></figcaption></figure>

5. Укажите **стоимость сделки** \
   Это значение проставится в поле "Бюджет" в сделке amoCRM. Вы можете задать конкретное значение или использовать [переменные](../../peremennye). Поле является опциональным и необязательно для заполнения. &#x20;

### Информация о контакте

Вы можете прикрепить к сделке данные о контакте, а именно:

* 👤 Имя и Фамилия
* ☎️ Телефон&#x20;
* 📩 Email

Для заполнения вы также можете использовать переменные.

### Дублирование контакта&#x20;

Есть 2 причины дублирования контактов в amoCRM. Давайте рассмотрим каждую из них.

1. Удалили пользователя из Smartbot Pro

Если удалить пользователя из Smartbot Pro, но не удалять его контакт в amoCRM, то при повторном прохождении пользователем блока "Создать сделку", будут обновлены значения переменных `amocrm_contact_id` (id контакта из amoCRM) и `smartbotpro_user_id` (id контакта из Smartbot Pro). В итоге в списке контактов будут два похожих контакта с разными smartbotpro\_user\_id.

2. Несколько контактов одинаковые, но их источники отличаются

Например, к вашему сценарию привязаны два разных канала - Вконтакте и Telegram. Один и тот же пользователь дважды прошел сценарий, в котором был блок "Создать сделку". В списке каналов вы увидите два контакта с одинаковым номером телефона, но если зайдете внутрь контакта, то увидите, что значения переменной `smartbotpro_user_id` отличаются.

Также может быть случай, когда у одного контакта переменная `smartbotpro_user_id` есть, а у второго - нет. Это значит, пользователь пришел через канал, привязанный к Smartbot Pro, и через другой канал, несвязанный со Smartbot Pro.

Поэтому чтобы проверить уникальность пользователя и его контакта, посмотрите  на  значения переменной `smartbotpro_user_id`.&#x20;

В amoCRM есть функционал контроля дублей по нескольким параметрам ([подробнее тут](https://www.amocrm.ru/support/incoming_leads/kontrol-dublej)):

* по BIG DATA;
* по Пикселю;
* по email;
* по номеру телефона;
* по значению полей (сущностей Сделка или Контакт, с типом Текст и Число).

Если будут какие-то вопросы, обращайтесь в нашу техподдержку!
