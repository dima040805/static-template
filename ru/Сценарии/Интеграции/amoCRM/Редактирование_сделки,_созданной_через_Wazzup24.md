# Редактирование сделки, созданной через Wazzup24

Если у вас есть аккаунт wazzup и подключенная к нему CRM-система, то вы знаете, что при первом контакте пользователя с вами, сделка в CRM создается автоматически. Но, для актуализации сделки в процессе работы с клиентом, пришлось бы вручную менять ее параметры. В Smartbot можно сделать так, чтобы изменение статуса (или другого параметра) сделки происходило автоматически в связи с некоторым целевым событием в боте.&#x20;

{% note info %}

**Когда нужно изменять сделку автоматически**

Представим, что вы продаете некоторые товары, например, матрасы, и ваш бизнес-процесс устроен следующим образом: на первом этапе коммуникации диалог с клиентом ведет бот [в режиме AI](../../../instrumenty/smartbot-ai) - отвечает на вопросы и помогает определиться с выбором. Но, после того, как пользователь оставит номер телефона, необходимо перевести диалог на менеджера для персональной работы с клиентом. Для этого сделка должна перейти, например, из колонки "Первичный контакт" (диалоги из сделок в этом статусе отрабатывает бот) в статус "Оставил телефон" -- тогда менеджер сразу увидит изменение по сделке и возьмет заявку в работу.

{% endnote %}

### Настройки в кабинете Wazzup24

Перед тем, как перейти к настройке этой механики в Smartbot, убедитесь, что в вашем кабинете Wazzup24:

1. Настроена интеграция с amoCRM. \
   Если еще нет, то вот [инструкция, как это сделать](https://wazzup24.ru/help/amocrm/snachala-podkljuchite-integraciju-2/). После того, как выполните эти шаги, нужно выбрать, в какой воронке и в каком статусе должны создаваться сделки в Wazzup при первом контакте с пользователем.\
   ![](https://4058588211-files.gitbook.io/~/files/v0/b/gitbook-x-prod.appspot.com/o/spaces%2FvTd8XmFMqkqZga7zhLuk%2Fuploads%2Fr6iawOkhiZ1zfczpgrSm%2Fimage.png?alt=media\&token=4e960352-b432-4405-8324-1d8adcbc1e6c)

{% note info %}

Кстати, у статуса "Неразобранное" есть особенность - сделка будет создана пустой, т.е. информации об ответственном и контакте не будет. Ответственный в amoCRM проставится, как только менеджер в wazzup возьмет на себя диалог, но контакт заполняться не будет. Поэтому рекомендуем выбирать другие статусы, чтобы не терять информацию о пользователе!

{% endnote %}

2\. Подключены каналы коммуникации с пользователями (например, WhatsApp, Авито и другие). Если что, вот [инструкция](https://wazzup24.ru/help/how-to-configurate/). &#x20;

Если эти условия выполнены, можно переходить к настройке в Smartbot.&#x20;

### Настройка в кабинете Smartbot

1. Подключите канал Wazzup к смартботу, если еще не сделали этого. Это легко, рассказали подробно тут👉[podklyuchenie-wazzup](../../../nachalo-raboty/kak-podklyuchit-messendzher/podklyuchenie-wazzup "mention");
2. Подключите к Smartbot интеграцию **с тем аккаунтом amoCRM, который подключен к вашему кабинету Wazzup24** [по инструкции](). Интеграция может отобразиться не сразу, но после нескольких попыток все только получится!
3. &#x20;Свяжите кабинет Wazzup24 и amoCRM в разделе "Интеграции". Остановимся на этом шаге подробнее ниже 👇
4. Последним шагом останется настроить изменение конкретных полей в сделке amoCRM через [blok-izmenit-sdelku](blok-izmenit-sdelku "mention")

### Привязка wazzup24 к amoCRM в Smartbot

Рассмотрим подробнее шаг 3 из инструкции выше. Связка кабинетов Wazzup24 и amoCRM нужна для того, чтобы бот понимал, в каком кабинете amoCRM создаются сделки из конкретного канала wazzup24, поэтому нам потребуется это указать явно в настройках интеграции amoCRM в Smartbot Pro. Особенно это важно для случаев, если у вас есть несколько привязанных к Smartbot аккаунтов amoCRM.&#x20;

Перейдите в разделе "Интеграции" -> "amoCRM". Выберите тот же аккаунт amo, который подключен к вашему кабинету Wazzup24, разверните его настройки и в поле для выбора канала выберите канал Wazzup.

{% note tip %}

Поздравляем! Вы создали связку между своими кабинетами wazzup и amocrm!

{% endnote %}

Теперь вы можете настроить [blok-izmenit-sdelku](blok-izmenit-sdelku "mention"), чтобы изменять воронку, статус или другие параметры сделки, созданной автоматически Wazzup24.&#x20;

Для этого в поле "ID сделки" укажите переменную, которая содержит в себе id сделки в amoCRM из wazzup. Это может быть специальная переменная %wazzup\_lead\_id% или ваша переменная, в которую вы пересохраняете значение первой.

{% note info %}

Обратите внимание, что сделка с пользователем создается только при его **первом контакте** с wazzup24!&#x20;

Если пользователь уже писал в какой-то из ваших каналов, подключенных к wazzup24, новая сделка для него не создастся и переменная %wazzup\_lead\_id% будет пустой. Лучше пересохраните значения %wazzup\_lead\_id% в другую пользовательскую переменную и руками заполните значения уже имеющихся сделок в amoCRM.

{% endnote %}

{% note info %}

Если в вашем сценарии сделки создаются не только через Wazzup, но и через блок "Создать сделку", при настройке блока "Изменить сделку" внимательно посмотрите, какой id сделки используете.

{% endnote %}
