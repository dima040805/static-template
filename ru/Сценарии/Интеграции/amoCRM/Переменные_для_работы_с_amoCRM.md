# Переменные для работы с amoCRM

На данный момент есть три переменные:

1. %amocrm\_lead\_id%
2. %amocrm\_contact\_id%
3. smartbotpro\_user\_id

Далее поговорим подробнее о каждой ✨

**%amocrm\_lead\_id%**

`%amocrm_lead_id%` - специальная переменная, которая хранит идентификатор **последней созданной** для пользователя сделки. Если пользователь пройдёт несколько раз блок "Создать сделку", то переменная будет хранить последнее значение. \
Если необходимо зафиксировать за пользователем конкретную сделку, рекомендуем пересохранить нужное значение в пользовательскую переменную.&#x20;

**%amocrm\_contact\_id%**

`%amocrm_contact_id%` - специальная переменная, которая хранит id контакта, с которым связана сделка.&#x20;

{% note info %}

Если к вашему сценарию подключены разные каналы, то есть вероятность, что один и тот же человек напишет вам из разных каналов. В таком случае в разделе "Пользователи" это будут два разных пользователя и для каждого из них будет свое значение переменной %amocrm\_contact\_id%.

{% endnote %}

**smartbotpro\_user\_id**

Эта переменная также хранит в себе айди пользователя, но в данном случае - это id пользователя со стороны Smartbot Pro. `smartbotpro_user_id` **не** является специальной переменной, то есть с этим значением нельзя работать внутри бота. \
`smartbotpro_user_id` подгружается в контакт и сделку и нужен для того, чтобы определять в amoCRM сделки, созданные с помощью Smartbot. Также по значению этой переменной вы можете определить уникальность контакта на стороне Smartbot - если значение этой переменной в нескольких сделках одинаковое, значит сделка принадлежит одному и тому же пользователю, пришедшему из бота.&#x20;
