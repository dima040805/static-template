# Добавление менеджеров

{% note info %}

Есть два уровня доступа: _менеджер **кабинета**_ и _менеджер **проекта**. Роли можно комбинировать между собой для гибкой настройки прав. Например, менеджер может иметь права гостя в кабинете и доступ на редактирование в конкретном проекте._

{% endnote %}

{% note info %}

Напомним: **проект** - это сборник сценариев и каналов, подключенных к ним, а **кабинет** - объединяет в себе все проекты. &#x20;

{% endnote %}

Если вашем кабинете есть несколько разных проектов, и вы можете:\
а) выдать пользователю права в рамках всего кабинета, и тогда у него будет доступ ко всем проектам;\
б) выдать пользователю доступ только к одному или нескольким конкретным проектам, а доступ к остальным - запретить.

Все пользователи сервиса, независимо от роли, могут:

1. просмотреть список менеджеров кабинета с ролью и почтой;
2. обратиться в техническую поддержку с вопросом;
3. присоединиться к партнерской программе (кстати, вот ссылка на бота [https://t.me/partnership\_smartbot](https://t.me/partnership_smartbot));
4. отправить заявку на заказ бота под ключ.&#x20;

### Добавление менеджера кабинета

Для добавления менеджера **в кабинет** владелец кабинета должен зайти в настройки кабинета и выбрать функцию "Добавить".&#x20;

Понадобится ввести почту нового менеджера и выбрать его роль в кабинете. Менеджер получит на почту приглашение, в которое будет вложена ссылка на форму регистрации.&#x20;

<figure><img src="https://4058588211-files.gitbook.io/~/files/v0/b/gitbook-x-prod.appspot.com/o/spaces%2FvTd8XmFMqkqZga7zhLuk%2Fuploads%2FrC6DcrqvQnJXgdKaJ4nQ%2Fimage.png?alt=media&#x26;token=3fecac67-6bfb-48f7-bc51-0ce591e72134" alt=""><figcaption></figcaption></figure>

После регистрации менеджер попадет в нужный кабинет.&#x20;

#### Роли менеджеров в кабинете

Каждая последующая роль включает все права предыдущей, дополняя их новыми возможностями. Это значит, что роли строятся по принципу накопления: новые роли наследуют предыдущие права и приобретают дополнительные функции:

{% note info %}

**Гость** - права в кабинете отсутствуют. Это роль, которая выдается менеджеру по умолчанию, когда мы добавляем его только в конкретный проект с доступом на просмотр или редактирование. Такой менеджер не увидит список проектов кабинета, в которые его не добавили.&#x20;

**Наблюдатель** - по умолчанию может просматривать все проекты кабинета и сценарии без возможности изменить что-либо: текст, настройки, каналы и т.д. Доступен **только** просмотр.&#x20;

**Редактор** - по умолчанию может просматривать все проекты кабинета, редактировать и публиковать сценарии, а также работать с настройками проекта - например, включить и выключить отладку, добавить канал;

**Администратор** - управляет всеми проектами кабинета: настройками, менеджерами, а также может добавить и/или удалить проект.&#x20;

**Владелец -**  может добавлять новых менеджеров кабинета и менять их статус

{% endnote %}

Также есть особая роль **"Управляющий балансом"** - менеджер с этой ролью имеет доступ только к разделу "Баланс" в кабинете, он управляет бюджетом.&#x20;

Владелец также может потом изменить роль менеджера в том же разделе настроек кабинета.

### Добавление менеджера проекта

Чтобы сделать человека менеджером проекта заходим в настройки конкретного **проекта** на вкладку "Менеджеры" и вводим почту:&#x20;

<figure><img src="https://4058588211-files.gitbook.io/~/files/v0/b/gitbook-x-prod.appspot.com/o/spaces%2FvTd8XmFMqkqZga7zhLuk%2Fuploads%2F0GwrMy2I4G0NFpXPmTph%2Fimage.png?alt=media&#x26;token=1835dc5b-cd16-4e3e-ae37-f682236cc24d" alt=""><figcaption></figcaption></figure>

<figure><img src="https://4058588211-files.gitbook.io/~/files/v0/b/gitbook-x-prod.appspot.com/o/spaces%2FvTd8XmFMqkqZga7zhLuk%2Fuploads%2FtWDeJUh8PB1ZadGFhUdw%2Fimage.png?alt=media&#x26;token=09b1c672-ef35-46fa-81fd-31db9ae12e4e" alt=""><figcaption></figcaption></figure>

Когда менеджер получит письмо и пройдет по нему авторизацию, он сможет войти в кабинет с доступом к нужному проекту.

Для проекта можно выбрать доступы:&#x20;

* Редактирование и просмотр
* Только просмотр

После того, как добавили менеджера проекта, он появится в менеджерах **кабинета** с правами "Гость", то есть у него будут права **только** на нужном проекте.

### Кейсы и решения

* Если надо сделать человека **менеджером с&#x20;**_**одинаковыми правами во всех проектах кабинета**_, то делаем его **менеджером кабинета** с определенной ролью;
* Если необходимо дать права на _редактирование всех проектов кабинета, **кроме одного,**_ то назначаем его **менеджером кабинета** с ролью Администратор или Редактор, а в настройках конкретного проекта изменяем его права на "Только просмотр";
* Если нам нужно добавить менеджера только в один проек&#x442;_**,**_ то приглашаем его в качестве **менеджера кабинета** с ролью "Гость", а в нужном проекте поднимаем его полномочия до "Редактирования и просмотра".

{% note tip %}

Теперь вы знаете, как назначать менеджеров с нужным уровнем доступа! Если у вас остались вопросы, не стесняйтесь задать их нашей технической поддержке 🤝

{% endnote %}
