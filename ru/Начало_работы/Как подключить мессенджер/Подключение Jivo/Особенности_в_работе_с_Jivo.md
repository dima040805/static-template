# Особенности в работе с Jivo

В связи с тем, что Jivo является агрегатором каналов, есть некоторые ограничения, влияющие на работу бота в подключенных через Jivo каналах. Давайте рассмотрим каждое из них, чтобы учесть это при создании сценария.

1. **Jivo не поддерживает работу inline кнопок**.\
   Если в блоке "Отправить сообщение" поставить настройку "Показывать кнопки внутри сообщения (inline)", она придет в виде обычной кнопки. Поэтому рекомендуем изначально использовать обычные кнопки, чтобы это не было для вас сюрпризом!

2. **В каналы Jivo нельзя отправить рассылку.**\
   Тут все понятно из названия - для jivo не поддерживается массовая отправка рассылок по подключенным каналам.
3. **Бот может писать пользователю в течение 2х часов с момента последнего сообщения пользователя.**\
   Что за двухчасовое окно? Если за это время в диалоге от пользователя не будет сообщений, диалог автоматически закроется, и бот не сможет написать пользователю снова. Как только этот же пользователь напишет боту, то создастся новый диалог, боту вернется возможность писать, НО также создастся и новый пользователь в списке. Учитывайте эту особенность при создании сценария и сборе данных с пользователя!
4. **В диалоге одновременно может быть либо оператор, либо бот.**\
   Если клиенту первым напишет оператор **из Jivo**, не из Smartbot Pro, то бот в этом диалоге участвовать не будет, так как в диалоге может участвовать либо бот, либо оператор – но не оба одновременно. Бот в Jivo подключается только к тем диалогам, которые инициированы самим клиентом. Чтобы бот смог подключиться к чату **после сообщения от пользователя**, оператору нужно просто выйти из диалога.
